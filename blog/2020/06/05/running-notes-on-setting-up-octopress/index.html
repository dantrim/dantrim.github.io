Deploying Blog Entries after a Fresh Clone

If you checkout your blog's repository in a new location (e.g. a new directory or a new machine), you will first need to sync up the _deploy directory with the remote's master branch before running rake deploy. Run the following steps to achieve this:

1 $ git clone git@github.com:<repo>.git
2 $ cd <repo>
3 $ git checkout source # source is the working branch
4 $ mkdir _deploy
5 $ cd _deploy
6 $ git init
7 $ git remote add origin git@github.com:<repo>.git
8 $ git pull origin master # master is the public view of the blog
9 $ cd ..
